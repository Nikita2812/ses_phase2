**To the Backend & Frontend Engineers:** This is **Part 8 of 15** of the Technical Implementation & Domain Specification. This document provides the **Data Contract Catalog**, a central registry of all JSON schemas used for communication between different modules of the AIaaS platform. Adhering to these contracts is critical for ensuring system stability and interoperability.

# Tech Spec 8: Data Contract Catalog

**Version:** 1.0 (Implementation-Ready) **Audience:** Backend Engineers, Frontend Engineers, AI Engineers

## 1\. Data Contract: IsolatedFoundationInput

*   **Purpose:** The input data required to start the Isolated Foundation design workflow.
*   **Producer:** Frontend UI (User Input)
*   **Consumer:** civil\_foundation\_designer\_v1.py

{

"$schema": "http://json-schema.org/draft-07/schema#",

"title": "IsolatedFoundationInput",

"type": "object",

"properties": {

"column\_load\_p": { "type": "number", "description": "Axial load from the column in KN" },

"column\_size\_x": { "type": "number", "description": "Column dimension in X direction in mm" },

"column\_size\_y": { "type": "number", "description": "Column dimension in Y direction in mm" },

"soil\_bearing\_capacity\_sbc": { "type": "number", "description": "Safe Bearing Capacity of the soil in KN/m^2" },

"concrete\_grade\_fck": { "type": "string", "enum": \["M20", "M25", "M30"\] },

"steel\_grade\_fy": { "type": "string", "enum": \["Fe415", "Fe500"\] }

},

"required": \["column\_load\_p", "column\_size\_x", "column\_size\_y", "soil\_bearing\_capacity\_sbc", "concrete\_grade\_fck", "steel\_grade\_fy"\]

}

## 2\. Data Contract: IsolatedFoundationOutput

*   **Purpose:** The output data generated by the Isolated Foundation design workflow.
*   **Producer:** civil\_foundation\_designer\_v1.py
*   **Consumer:** Frontend UI (for display), Database (for storage)

{

"$schema": "http://json-schema.org/draft-07/schema#",

"title": "IsolatedFoundationOutput",

"type": "object",

"properties": {

"footing\_size\_x": { "type": "number" },

"footing\_size\_y": { "type": "number" },

"footing\_depth": { "type": "number" },

"reinforcement\_schedule\_x\_dir": { "type": "string" },

"reinforcement\_schedule\_y\_dir": { "type": "string" },

"shear\_check\_status": { "type": "string", "enum": \["PASS", "FAIL"\] },

"bending\_check\_status": { "type": "string", "enum": \["PASS", "FAIL"\] }

}

}

## 3\. Data Contract: PipingLoadInput

*   **Purpose:** The data contract for receiving pipe rack load information from the Piping discipline.
*   **Producer:** Piping Discipline (via API or file import)
*   **Consumer:** structural\_piperack\_designer\_v1.py

{

"$schema": "http://json-schema.org/draft-07/schema#",

"title": "PipingLoadInput",

"type": "object",

"properties": {

"rack\_id": { "type": "string" },

"bay\_number": { "type": "number" },

"loads": {

"type": "array",

"items": {

"type": "object",

"properties": {

"load\_type": { "type": "string", "enum": \["gravity", "thermal", "wind"\] },

"fx": { "type": "number", "description": "Force in X direction in KN" },

"fy": { "type": "number", "description": "Force in Y direction in KN" },

"fz": { "type": "number", "description": "Force in Z direction in KN" }

}

}

}

}

}

This catalog must be expanded to include a data contract for every single input and output in the entire AIaaS system. This ensures that all modules communicate with each other in a predictable and validated way.
